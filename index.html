Introduction

Some good points about grid

  • Its main properties do what they say on the tin
  • It's easy to use
  • Can make responsive layouts without the need for Bootstrap grid or even media queries (more on that later)
  • Excellent cross-browser compatibility, including IE (more on that later)

CSS grid is a layout framework that creates a grid, with rows and columns, allowing for easy layouts, without having to use position, floats, or Bootstrap grid. While flex display is used for single-direction content flow (x or y axis), grid is for multi-direction content flow (x and y).

Give a grid-container the display attribute grid/inline-grid, and its immediate children will become grid items. These will be placed according to the grid-template attributes specified.

In the rows/columns template attribute, you can specify a number of value types: fixed pixel widths, fluid percentile widths, auto width, or segment width.

auto

Leaving the value as auto means the dimension of the row/column is determined by the box content of the grid item. Therefore items with more cotent take up more space, while items with less content will render smaller.

Segment: fr

The fr measure means a fraction, like a division.

The client takes the total number of fr specified in the column/row, divides the space, and assigns the portion specified for the child in that area of the grid. So a grid-template-columns of 1fr 3fr 2fr will divide the space into 6, place the first child in the first space taking up one space, the second child in the second space and spanning 3 columns, and the third child spanning the final 2 columns.

Grid with 5 1fr-width items

With 5 1fr items specified in the template columns, the grid is made of 5 equal segments. The segments expand to fill available space in the container.

auto-fit, auto-fill

Specify repeat(), fit or fill, then a minmax.

.grid-syntax {
   grid-template-columns: // repeat([auto-fit/auto-fill], minmax([min-width],[max-width]));
}
.grid-container {
   grid-template-columns: repeat(auto-fit, minmax(330px,1fr));
}

The difference between auto-fill and auto-fit is a bit like the difference between an auto and 1fr. At smaller-container widths, the difference isn't appreciable, but it is at wider-container widths.

The auto-fill with minmax(100px, 1fr) will keep placing 100px-wide grid items, but will not make the grid items wider than the specified max width in order to fill excess space.

IE fallbacks

There is one thing you can say for Internet Explorer: it certainly exists.

It does, however, have pretty good support for grid; IE has always had an innate grid.

But

In this case, IE's interesting little foible is that it's a bit confused about where to place the grid items. As in it will place all of them at 1:1. You have to specify where each individual one goes. All of them.

This needs to be done with IE-specific CSS attributes, so you can easily do this without disturbing the layout in other, less rubbish browsers.

Generic syntax IE syntax
display: grid; display: -ms-grid;
grid-column -ms-grid-column
grid-row -ms-grid-row
grid-column: 1 / span 3; -ms-grid-column: 1; -ms-grid-column-span: 3;
